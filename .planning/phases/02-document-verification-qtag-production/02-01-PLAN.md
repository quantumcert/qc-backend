<threat_model>
## Trust Boundaries

- Boundary A: usuario anonimo da internet -> `GET /api/v1/public/verify/document/:hash`.
- Boundary B: tenant autenticado/qc-record-module -> `POST /api/v1/diamond` com `event.recordAuthenticated`.
- Boundary C: dados privados de tenant/asset -> payload publico de verificacao.
- Boundary D: schema do banco -> idempotencia de `documentHash`.

## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-02-01 | high | rota publica alternativa ou autenticada causa contrato divergente e quebra integracao publica | Manter somente `/api/v1/public/verify/document/:hash`; teste grep bloqueia rota alternativa |
| T-02-02 | high | duplicatas de `documentHash` por tenant retornam provas ambiguas | `@@unique([tenantId, documentHash])` + retorno idempotente em `EventLogFacet` |
| T-02-03 | high | payload publico vaza metadata, tenantId, owner, assinatura ou segredo | Selects explicitos e testes negativos de campos proibidos |
| T-02-04 | medium | hash invalido vira 404 e dificulta diagnostico/abuso | `INVALID_DOCUMENT_HASH` com HTTP 400 estruturado |
| T-02-05 | medium | documento ausente sem code impede consumidores de distinguir erro | `DOCUMENT_NOT_FOUND` com HTTP 404 estruturado |
</threat_model>
