<threat_model>

## Trust Boundaries

- Boundary A: anonymous HTTP client -> optional document payment hook.
- Boundary B: future payment provider -> backend route authorization.
- Boundary C: public document proof -> sensitive tenant/event data.

## Threats

- **HIGH: Accidental payment enable without provider.** Mitigation: default `X402_ENABLED=false`; when true without provider, fail closed with `PAYMENT_PROVIDER_NOT_CONFIGURED`.
- **HIGH: Sensitive data exposure through new `blockchain` object.** Mitigation: only expose tx id, explorer URL, chain, and anchoredAt; keep existing forbidden-field tests.
- **MEDIUM: Payment config accidentally applied globally.** Mitigation: route-scoped middleware and test `/api/v1/scan` remains ungated.
- **MEDIUM: Product/provider drift.** Mitigation: do not install provider packages yet; document Anchor/BRZ direction for later.
  </threat_model>
