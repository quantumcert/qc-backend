<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cliente HTTP → CircuitBreaker route | ADMIN externo envia assinatura Falcon-512 que precisa ser verificada antes de pausar a plataforma |
| Filesystem env (.env) → KMSService | Secret material é lido do ambiente — leak compromete chave master |
| QuantumSigner caller (Facets) → verifySignature | Múltiplos call sites assumem que verifySignature é honesto; um stub `return true` é equivalente a aceitar qualquer assinatura |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | CircuitBreakerService.verifyAdminSignature | mitigate | Substituir aceitação de "qualquer string não-vazia" por PostQuantumCrypto.verifySignatureFalcon512 contra CIRCUIT_BREAKER_ADMIN_PUBKEY |
| T-01-02 | Tampering | QuantumSignerService.verifySignature stub | mitigate | Remover `return true`; delegar para PostQuantumCrypto.verifySignatureFalcon512 |
| T-01-03 | Repudiation | KMSService chave Falcon efêmera | mitigate | Fail-fast em NODE_ENV=production se QUANTUM_CERT_SECRET ausente; documentar geração em .env.example |
| T-01-04 | Information Disclosure | CIRCUIT_BREAKER_ADMIN_PUBKEY no .env.example | accept | É uma public key — não há leak de secret; exemplo é placeholder vazio |
| T-01-05 | Denial of Service | Throw síncrono em getQuantumMasterKey() pode crashar requests inflight | accept | O throw só ocorre em startup (lazy init via REQUIRED_ENV_VARS) — não em runtime após boot |
</threat_model>
