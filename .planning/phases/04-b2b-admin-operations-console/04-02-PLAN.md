<threat_model>

## Trust Boundaries

- Boundary A: Platform Admin UI -> dashboard tRPC -> backend admin contract.
- Boundary B: tenant lifecycle status -> API key/access enablement.
- Boundary C: tenant commercial metadata -> audit and billing foundation.

## Threats

- **HIGH: Tenant criado/ativado sem autorizacao Quantum.** Mitigation:
  `requirePlatformAdmin` no backend e `platformAdminProcedure` no dashboard.
- **HIGH: Status comercial diverge de `isActive`.** Mitigation:
  lifecycle facet atualiza status, timestamps e compat field na mesma transacao.
- **HIGH: UI permite acao sem audit.** Mitigation:
  todas mutations exigem `reason`, actor e gravam `AdminAuditLog`.
- **MEDIUM: Tenant Admin enxerga tenants alheios.** Mitigation:
  Tenant Admin procedures usam membership/tenant atual; testes negam cross-tenant.
- **MEDIUM: Layout admin vira marketing ou mock.** Mitigation:
  seguir `04-UI-SPEC.md`: tabelas, tabs, estados operacionais e sem hero.
</threat_model>
