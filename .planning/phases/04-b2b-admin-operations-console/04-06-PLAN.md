<threat_model>

## Trust Boundaries

- Boundary A: dashboard local DB -> backend canonical user store.
- Boundary B: loose ownerRef -> canonical TenantUser link.
- Boundary C: migration execution -> existing production data.
- Boundary D: dashboard cutover -> backend contracts.

## Threats

- **HIGH: Usuario B2C vira tenant indevidamente.** Mitigation:
  `ensureTenantQuantum` e memberships sob Tenant Quantum; testes de contrato.
- **HIGH: Backfill duplica ou sobrescreve usuario errado.** Mitigation:
  upsert por chaves legacy/naturais seguras, conflict report e idempotency keys.
- **HIGH: OwnerRef associado ao usuario errado.** Mitigation:
  reconciliar somente matches confiaveis; preservar `Owner.ownerRef` legado.
- **HIGH: Credit balance migra para wallet financeira.** Mitigation:
  usar `CreditLedgerEntry`; testes preservam "nao mexer no saldo, somente no credito".
- **MEDIUM: Cutover executado antes de report aprovado.** Mitigation:
  script dry-run/execute separado, checkpoints e gate operacional documentado.
</threat_model>
