<threat_model>

## Trust Boundaries

- Boundary A: integrated dashboard routes -> backend admin contracts.
- Boundary B: Platform Admin cross-tenant -> Tenant Admin own-tenant.
- Boundary C: automated verification -> human operational backfill/QTAG UAT.

## Threats

- **HIGH: Tenant Admin burla tenant scope por parametro.** Mitigation:
  tRPC/backend ignoram tenantId arbitrario e usam contexto/membership; testes
  tentam injection.
- **HIGH: Phase 4 marcada pronta sem backfill aprovado.** Mitigation:
  `04-HUMAN-UAT.md` exige dry-run, execute e relatorio antes de closure.
- **HIGH: Admin UI integrada mas sem server checks.** Mitigation:
  smoke cross-repo chama procedures protegidas e backend auth.
- **MEDIUM: Fila operacional mascara fallback/mock.** Mitigation:
  fallback deve ser marcado local/demo; contracts reais obrigatorios quando
  backend existir.
- **MEDIUM: Dependencia fisica QTAG bloqueia encerramento sem visibilidade.**
  Mitigation: UAT registra se `qc-record-module`/writer fisico bloqueia a
  validacao final, separado da entrega de software.
</threat_model>
